# Security Policy

## Supported Versions

Security fixes are expected to land on the current development branch and the latest shipped mobile build line maintained by the team.

## Reporting a Vulnerability

Do not open a public issue for security-sensitive reports.

Instead:

1. Contact the maintainers through the private security channel already used by your organization or deployment team.
2. Include the affected platform, app version, impact, reproduction steps, and any proof-of-concept details.
3. State whether the issue is already known to any third party.

If you do not have a private channel yet, open a minimal issue requesting a private contact path without disclosing exploit details.

## Response Expectations

Maintainers should acknowledge receipt, confirm impact, and coordinate remediation before public disclosure.
